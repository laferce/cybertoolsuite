OneForAll 帮助文档

描述

OneForAll 是一个功能强大的子域名整合工具。

用法

python oneforall.py [组]|[命令]|[值]
示例
  • python oneforall.py version
  • python oneforall.py check
  • python oneforall.py --target example.com run
  • python oneforall.py --targets ./domains.txt run
  • python oneforall.py --target example.com --alive False run
  • python oneforall.py --target example.com --brute False run
  • python oneforall.py --target example.com --port medium run
  • python oneforall.py --target example.com --fmt csv run
  • python oneforall.py --target example.com --dns False run
  • python oneforall.py --target example.com --req False run
  • python oneforall.py --target example.com --takeover False run
  • python oneforall.py --target example.com --show True run
注意
  • --port 详情请参阅 ./config/default.py ./config/setting.py（默认值：small；可选：small、medium、large）
  • --fmt 结果格式（默认值：csv；可选：csv、json）
  • --path 结果路径（未配置时自动生成）
组
  • data：数据。
  • datas：多个数据。
  • domains：多个域名。
命令
  • check：检查是否有新版本并退出。
  • check_param：检查参数。
  • config_param：配置参数。
  • export_data：从数据库导出数据。
  • main：OneForAll 主进程。
  • run：OneForAll 运行入口。
  • version：打印版本信息并退出。
选项
  • --target=TARGET：单个域名（必须提供 target 或 targets）。
  • --targets=TARGETS：每行一个域名的文件路径。
  • -b, --brute=BRUTE：使用暴力模块（默认值：True）。
  • -d, --dns=DNS：使用 DNS 解析（默认值：True）。
  • -r, --req=REQ：HTTP 请求子域名（默认值：True）。
  • --port=PORT：请求的端口范围（默认 small 端口为 80,443）。
  • -a, --alive=ALIVE：仅导出存活的子域名（默认值：False）。
  • -f, --fmt=FMT：结果格式（默认值：csv）。
  • --path=PATH：结果路径（未配置时自动生成）。
  • --takeover=TAKEOVER：扫描子域名接管（默认值：False）。
值
  • access_internet：网络连通性检测。
  • alive：存活检测。
  • brute：暴力破解。
  • dns：DNS 解析。
  • domain：域名处理。
  • enable_wildcard：启用泛解析处理。
  • fmt：输出格式。
  • in_china：国内网络环境。
  • path：路径设置。
  • port：端口扫描。
  • req：HTTP 请求。
  • takeover：子域名劫持检测。
  • target：单个目标。
  • targets：目标列表文件。
