docker 帮助文档

用法

docker [选项] 命令 [参数...]
全局选项
  • --config=%USERPROFILE%\.docker：客户端配置文件的位置（默认用户目录下的 .docker 文件夹）。
  • -D, --debug=false：启用调试模式（默认值：false）。
  • -H, --host=[]：连接的守护进程套接字（例如：tcp://192.168.1.1:2375）。
  • -l, --log-level=info：设置日志级别（例如：debug、info、warn、error）。
  • --tls=false：启用 TLS 加密（若启用，--tlsverify 自动生效）。
  • --tlscacert=%USERPROFILE%\.docker\ca.pem：信任由此 CA 签名的证书。
  • --tlscert=%USERPROFILE%\.docker\cert.pem：TLS 证书文件路径。
  • --tlskey=%USERPROFILE%\.docker\key.pem：TLS 密钥文件路径。
  • --tlsverify=false：启用 TLS 并验证远程服务器。
  • -v, --version=false：显示版本信息并退出。
子命令
  • 容器操作
    • attach：附加到正在运行的容器。
    • cp：将文件/文件夹从容器复制到宿主机目录或标准输出。
    • create：创建新容器。
    • diff：检查容器文件系统的变更。
    • exec：在运行中的容器内执行命令。
    • export：将容器文件系统导出为 tar 归档文件。
    • kill：强制停止运行中的容器。
    • logs：获取容器的日志。
    • pause：暂停容器内所有进程。
    • port：列出容器的端口映射。
    • ps：列出容器。
    • rename：重命名容器。
    • restart：重启容器。
    • rm：删除一个或多个容器。
    • run：在新容器中运行命令。
    • start：启动已停止的容器。
    • stats：实时显示容器资源使用统计。
    • stop：停止运行中的容器。
    • top：显示容器的运行中进程。
    • unpause：恢复容器内所有进程。
    • wait：阻塞直到容器停止，并打印退出码。
    镜像管理
    • build：从 Dockerfile 构建镜像。
    • commit：基于容器变更创建新镜像。
    • history：显示镜像的历史记录。
    • images：列出镜像。
    • import：从 tarball 导入内容以创建文件系统镜像。
    • inspect：获取容器或镜像的底层信息。
    • load：从 tar 归档或标准输入加载镜像。
    • pull：从镜像仓库拉取镜像。
    • push：将镜像推送到镜像仓库。
    • rmi：删除一个或多个镜像。
    • save：将镜像保存为 tar 归档。
    • search：在 Docker Hub 中搜索镜像。
    • tag：为镜像创建新标签。
    系统与仓库
    • events：从服务器获取实时事件。
    • info：显示系统级信息。
    • login：登录 Docker 镜像仓库。
    • logout：退出 Docker 镜像仓库。
    • version：显示 Docker 版本信息。
