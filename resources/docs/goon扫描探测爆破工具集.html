goon扫描探测爆破工具集 帮助文档

用法

goon3_win_amd64.exe [选项]
选项
  • -body string：DIR 请求包的 body，例如：{"name":"username"}。
  • -code int：DIR 返回包的 HTTP 状态码，例如：200、302（默认值：200）。
  • -dir string：DIR fuzz 请求的路径，例如：/login.jsp。支持 post 发包，支持正则匹配，可探测简单 PoC。
  • -dmode string：DIR fuzz 请求方式，支持 get 或 post（默认值：get）。
  • -fields string：FOFA 返回类型，例如：ip,host。
  • -header string：DIR 请求包的 Header，例如：Content-Type:Application/json。
  • -help：显示模式详解。
  • -ifile string：输入文件，可以是 URL、IP 或 IP 段。IP 段支持 /8 到 /31 的任意 CIDR。
  • -ip string：指定 IP 地址或 IP 段，例如：127.0.0.1、127.0.0.1/24、127.0.0.1-255。IP 段支持 /8 到 /31 的任意 CIDR。
  • -key string：FOFA 查询语句，例如：domain='fofa.so'。
  • -mode string：运行模式，例如：webscan、brute、title、fofa、mysql、mssql 等（默认值：all）。
  • -no：不输出扫描日志，避免在 Linux 系统下因文件权限问题报错。
  • -np：不进行 ICMP 探活。
  • -num int：FOFA 请求的数量，例如：100、10000。
  • -ofile string：输出文件（默认值：result.txt）。
  • -pass string：密码，默认从配置文件读取。
  • -pfile string：密码字典，默认从配置文件读取。
  • -ping：不进行 ICMP 探活，而是调用 ping 来探活，适用于非 root/admin 权限用户。
  • -port string：扫描端口，例如：80,443-445,8000-9000。
  • -rbody string：DIR 返回包的 body，例如：root:x:0:0，支持正则表达式。
  • -rheader string：DIR 返回包的 header，例如：rememberMe，支持正则表达式。
  • -svar string：一个字符串变量（默认值："bar"）。
  • -thread int：线程数，默认从配置文件读取。
  • -time int：超时时间，默认从配置文件读取。
  • -ufile string：用户字典，默认从配置文件读取。
  • -url string：目标 URL，必须包含 http。
  • -user string：用户名，默认从配置文件读取。
  • -web：Port 和 FOFA Host 输出格式，例如：http://127.0.0.1:80。
