Httpx辅助工具 帮助文档

用法

httpx.exe [选项]

选项说明

输入
  • -l, -list string：输入包含主机列表的文件。
  • -rr, -request string：包含原始请求的文件。
  • -u, -target string[]：输入要探测的目标主机。
探测
  • -sc, -status-code：显示响应状态码。
  • -cl, -content-length：显示响应内容长度。
  • -ct, -content-type：显示响应内容类型。
  • -location：显示响应重定向位置。
  • -favicon：显示 /favicon.ico 文件的 mmh3 哈希值。
  • -hash string：显示响应体哈希值（支持：md5, mmh3, simhash, sha1, sha256, sha512）。
  • -jarm：显示 jarm 指纹哈希。
  • -rt, -response-time：显示响应时间。
  • -lc, -line-count：显示响应体行数。
  • -wc, -word-count：显示响应体字数。
  • -title：显示页面标题。
  • -bp, -body-preview：显示响应体前 N 个字符（默认 100）。
  • -server, -web-server：显示服务器名称。
  • -td, -tech-detect：基于 wappalyzer 数据集显示使用的技术。
  • -method：显示 HTTP 请求方法。
  • -websocket：显示服务器是否使用 WebSocket。
  • -ip：显示主机 IP。
  • -cname：显示主机 CNAME。
  • -asn：显示主机 ASN 信息。
  • -cdn：显示使用的 CDN/WAF（默认开启）。
  • -probe：显示探测状态。
无头浏览器
  • -ss, -screenshot：使用无头浏览器保存页面截图。
  • -system-chrome：使用本地安装的 Chrome 进行截图。
  • -ho, -headless-options string[]：启动无头 Chrome 时附加选项。
  • -esb, -exclude-screenshot-bytes：从 JSON 输出中排除截图字节。
  • -ehb, -exclude-headless-body：从 JSON 输出中排除无头浏览器响应体。
  • -st, -screenshot-timeout int：设置截图超时时间（秒）。
匹配器
  • -mc, -match-code string：匹配指定状态码的响应。
  • -ml, -match-length string：匹配指定内容长度的响应。
  • -mlc, -match-line-count string：匹配指定行数的响应体。
  • -mwc, -match-word-count string：匹配指定字数的响应体。
  • -mfc, -match-favicon string[]：匹配指定图标哈希的响应。
  • -ms, -match-string string：匹配包含指定字符串的响应。
  • -mr, -match-regex string：匹配符合指定正则的响应。
  • -mcdn, -match-cdn string[]：匹配使用指定 CDN 提供商的主机。
  • -mrt, -match-response-time string：匹配指定响应时间的响应。
  • -mdc, -match-condition string：使用 DSL 表达式条件匹配响应。
提取器
  • -er, -extract-regex string[]：显示匹配正则的响应内容。
  • -ep, -extract-preset string[]：显示匹配预定义正则的响应内容。
过滤器
  • -fc, -filter-code string：过滤指定状态码的响应。
  • -fep, -filter-error-page：基于机器学习过滤错误页面。
  • -fl, -filter-length string：过滤指定内容长度的响应。
  • -flc, -filter-line-count string：过滤指定行数的响应体。
  • -fwc, -filter-word-count string：过滤指定字数的响应体。
  • -ffc, -filter-favicon string[]：过滤指定图标哈希的响应。
  • -fs, -filter-string string：过滤包含指定字符串的响应。
  • -fe, -filter-regex string：过滤符合指定正则的响应。
  • -fcdn, -filter-cdn string[]：过滤使用指定 CDN 提供商的主机。
  • -frt, -filter-response-time string：过滤指定响应时间的响应。
  • -fdc, -filter-condition string：使用 DSL 表达式条件过滤响应。
  • -strip：移除响应中的所有标签。
速率限制
  • -t, -threads int：使用的线程数。
  • -rl, -rate-limit int：每秒发送的最大请求数。
  • -rlm, -rate-limit-minute int：每分钟发送的最大请求数。
其他
  • -pa, -probe-all-ips：探测与同一主机关联的所有 IP。
  • -p, -ports string[]：探测的端口。
  • -path string：探测的路径或路径列表。
  • -tls-probe：在提取的 TLS 域上发送 HTTP 探测。
  • -csp-probe：在提取的 CSP 域上发送 HTTP 探测。
  • -tls-grab：执行 TLS(SSL) 数据抓取。
  • -pipeline：探测并显示支持 HTTP1.1 管线的服务器。
  • -http2：探测并显示支持 HTTP2 的服务器。
  • -vhost：探测并显示支持 VHOST 的服务器。
  • -ldv, -list-dsl-variables：列出支持 DSL 匹配器/过滤器的 JSON 输出字段键名。
更新
  • -up, -update：将 httpx 更新到最新版本。
  • -duc, -disable-update-check：禁用自动更新检查。
输出
  • -o, -output string：写入输出结果的文件。
  • -oa, -output-all：以所有格式写入输出结果的文件名。
  • -sr, -store-response：将 HTTP 响应存储到输出目录。
  • -srd, -store-response-dir string：将 HTTP 响应存储到自定义目录。
  • -csv：以 CSV 格式存储输出。
  • -csvo, -csv-output-encoding string：定义输出编码。
  • -j, -json：以 JSONL 格式存储输出。
  • -irh, -include-response-header：在 JSON 输出中包含 HTTP 响应头。
  • -irr, -include-response：在 JSON 输出中包含 HTTP 请求/响应。
  • -irrb, -include-response-base64：在 JSON 输出中包含 Base64 编码的 HTTP 请求/响应。
  • -include-chain：在 JSON 输出中包含重定向链。
  • -store-chain：在响应中包含 HTTP 重定向链。
  • -svrc, -store-vision-recon-cluster：包含视觉侦察集群。
配置
  • -config string：httpx 配置文件路径。
  • -auth：配置 projectdiscovery cloud (pdcp) API 密钥。
  • -r, -resolvers string[]：自定义解析器列表。
  • -allow string[]：允许处理的 IP/CIDR 列表。
  • -deny string[]：禁止处理的 IP/CIDR 列表。
  • -sni, -sni-name string：自定义 TLS SNI 名称。
  • -random-agent：启用随机 User-Agent。
  • -H, -header string[]：自定义 HTTP 头。
  • -http-proxy, -proxy string：使用的 HTTP 代理。
  • -unsafe：发送原始请求，跳过 Go 语言规范化。
  • -resume：使用 resume.cfg 恢复扫描。
  • -fr, -follow-redirects：跟随 HTTP 重定向。
  • -maxr, -max-redirects int：每个主机跟随重定向的最大次数。
  • -fhr, -follow-host-redirects：跟随同一主机的重定向。
  • -rhsts, -respect-hsts：尊重 HSTS 响应头进行重定向请求。
  • -vhost-input：将 vhosts 列表作为输入。
  • -x string：探测的请求方法。
  • -body string：HTTP 请求中包含的 POST 体。
  • -s, -stream：流模式 - 无需排序即可开始处理输入目标。
  • -sd, -skip-dedupe：禁用去重输入项（仅在流模式下使用）。
  • -ldp, -leave-default-ports：保留主机头中的默认 HTTP/HTTPS 端口。
  • -ztls：使用 ztls 库并自动回退到标准库进行 TLS1.3 处理。
  • -no-decode：避免解码响应体。
  • -tlsi, -tls-impersonate：启用实验性客户端 Hello (ja3) TLS 随机化。
  • -no-stdin：禁用标准输入处理。
调试
  • -health-check, -hc：运行诊断检查。
  • -debug：在命令行中显示请求/响应内容。
  • -debug-req：在命令行中显示请求内容。
  • -debug-resp：在命令行中显示响应内容。
  • -version：显示 httpx 版本。
  • -stats：显示扫描统计信息。
  • -profile-mem string：可选的 httpx 内存分析转储文件。
  • -silent：静默模式。
  • -v, -verbose：详细模式。
  • -si, -stats-interval int：显示统计信息更新的时间间隔。
  • -nc, -no-color：禁用命令行输出中的颜色。
优化
  • -nf, -no-fallback：显示探测的协议（HTTPS 和 HTTP）。
  • -nfs, -no-fallback-scheme：使用输入中指定的协议方案进行探测。
  • -maxhr, -max-host-error int：每个主机的最大错误数。
  • -e, -exclude string[]：排除匹配指定过滤器的主机。
  • -retries int：重试次数。
  • -timeout int：超时时间（秒）。
  • -delay value：HTTP 请求之间的间隔。
  • -rsts, -response-size-to-save int：保存的最大响应大小。
  • -rstr, -response-size-to-read int：读取的最大响应大小。
