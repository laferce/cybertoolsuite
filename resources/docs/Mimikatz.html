用法

mimikatz.exe
模块列表
  • standard：标准模块，包含基本命令，无需指定模块名称。
    • exit：退出 mimikatz。
    • cls：清屏（在重定向时无效，例如 PsExec）。
    • answer：回答生命、宇宙以及一切的终极问题。
    • coffee：请给我一杯咖啡！
    • sleep：休眠指定的毫秒数。
    • log：将 mimikatz 的输入/输出记录到文件。
    • base64：切换文件输入/输出为 Base64 编码。
    • version：显示版本信息。
    • cd：更改或显示当前目录。
    • localtime：显示系统本地日期和时间（OJ 命令）。
    • hostname：显示系统本地主机名。
  • crypto：加密模块，用于加密相关操作。
    • crypto::providers：列出加密服务提供者。
    • crypto::stores：列出加密存储。
    • crypto::certificates：列出（或导出）证书。
    • crypto::keys：列出（或导出）密钥容器。
    • crypto::sc：列出智能卡读卡器。
    • crypto::hash：使用可选的用户名对密码进行哈希处理。
    • crypto::system：描述 Windows 系统证书（文件、待定：注册表或 hive 文件）。
    • crypto::scauth：从 CA 创建身份验证证书（类似智能卡）。
    • crypto::certtohw：尝试将软件 CA 导出到加密（虚拟）硬件。
    • crypto::capi：[实验性] 修补 CryptoAPI 层以便于导出。
    • crypto::cng：[实验性] 修补 CNG 服务以便于导出。
    • crypto::extract：[实验性] 从 CAPI RSA/AES 提供者中提取密钥。
    • crypto::kutil：密钥实用工具。
    • crypto::tpminfo：显示 TPM 信息。
  • sekurlsa：SekurLSA 模块，用于枚举系统中的凭证。
    • sekurlsa::msv：列出 LM 和 NTLM 凭证。
    • sekurlsa::wdigest：列出 WDigest 凭证。
    • sekurlsa::kerberos：列出 Kerberos 凭证。
    • sekurlsa::tspkg：列出 TsPkg 凭证。
    • sekurlsa::livessp：列出 LiveSSP 凭证。
    • sekurlsa::cloudap：列出 CloudAp 凭证。
    • sekurlsa::ssp：列出 SSP 凭证。
    • sekurlsa::logonPasswords：列出所有可用提供者的凭证。
    • sekurlsa::process：切换（或重新初始化）到 LSASS 进程上下文。
    • sekurlsa::minidump：切换（或重新初始化）到 LSASS 的内存转储上下文。
    • sekurlsa::bootkey：设置 SecureKernel 启动密钥，以尝试解密 LSA 隔离凭证。
    • sekurlsa::pth：传递哈希（Pass-the-hash）。
    • sekurlsa::krbtgt：列出黄金票据。
    • sekurlsa::dpapisystem：DPAPI_SYSTEM 密钥。
    • sekurlsa::trust：列出钻石票据和跨域黄金票据。
    • sekurlsa::backupkeys：首选备份主密钥。
    • sekurlsa::tickets：列出 Kerberos 票据。
    • sekurlsa::ekeys：列出 Kerberos 加密密钥。
    • sekurlsa::dpapi：列出缓存的主密钥。
    • sekurlsa::credman：列出凭证管理器中的凭证。
  • kerberos：Kerberos 模块，用于处理 Kerberos 相关操作。
    • kerberos::ptt：传递票据（Pass-the-ticket），适用于 Windows NT 6.x 系统。
    • kerberos::list：列出票据。
    • kerberos::ask：请求或获取 TGS 票据。
    • kerberos::tgt：检索当前的 TGT（Ticket-Granting Ticket）。
    • kerberos::purge：清除票据。
    • kerberos::golden：生成 Golden Ticket（黄金票据）。
    • kerberos::hash：将密码哈希转换为密钥。
    • kerberos::ptc：传递 ccache 文件（Pass-the-ccache），适用于 Windows NT 6.x 系统。
    • kerberos::clist：列出 MIT/Heimdall ccache 文件中的票据。
  • ngc：Next Generation Cryptography 模块，仅限 kiwi 使用，用于枚举凭证。
    • ngc::logondata：:)
    • ngc::pin：尝试解密 PIN 保护器。
    • ngc::sign：尝试签名。
    • ngc::decrypt：尝试解密。
    • ngc::enum：枚举相关选项或信息。
  • privilege：权限模块，用于操作权限相关功能。
    • privilege::debug：请求调试特权。
    • privilege::driver：请求加载驱动程序特权。
    • privilege::security：请求安全特权。
    • privilege::tcb：请求 TCB（trusted computer base）特权。
    • privilege::backup：请求备份特权。
    • privilege::restore：请求还原特权。
    • privilege::sysenv：请求系统环境特权。
    • privilege::id：通过特权 ID 请求特权。
    • privilege::name：通过特权名称请求特权。
  • process：进程模块，用于操作和管理进程。
    • process::list：列出进程。
    • process::exports：列出导出项。
    • process::imports：列出导入项。
    • process::start：启动进程。
    • process::stop：终止进程。
    • process::suspend：挂起进程。
    • process::resume：恢复进程。
    • process::run：运行。
    • process::runp：运行程序。
  • service：服务模块，用于操作和管理服务。
    • service::start：启动服务。
    • service::remove：删除服务。
    • service::stop：停止服务。
    • service::suspend：挂起服务。
    • service::resume：恢复服务。
    • service::preshutdown：预关闭服务。
    • service::shutdown：关闭服务。
    • service::list：列出服务。
    • service::+：安装服务。
    • service::-：卸载服务。
    • service::me：显示当前服务信息。
  • lsadump：LsaDump 模块，用于转储 LSA 信息。
    • lsadump::sam：获取 SysKey 以解密 SAM 条目（从注册表或 hive 文件中提取）。
    • lsadump::secrets：获取 SysKey 以解密 SECRETS 条目（从注册表或 hive 文件中提取）。
    • lsadump::cache：获取 SysKey 以解密 NL$KM 和 MSCache(v2)（从注册表或 hive 文件中提取）。
    • lsadump::lsa：请求 LSA Server 以检索 SAM/AD 条目（正常模式、即时修补或注入）。
    • lsadump::trust：请求 LSA Server 以检索信任认证信息（正常模式或即时修补）。
    • lsadump::backupkeys：获取备份密钥。
    • lsadump::rpdata：获取 RDP 管理数据。
    • lsadump::dcsync：请求域控制器同步对象。
    • lsadump::dcshadow：他们告诉我可以成为任何我想成为的东西，所以我成了一个域控制器。
    • lsadump::setntlm：请求服务器为一个用户设置新的密码/NTLM 哈希。
    • lsadump::changentlm：请求服务器为一个用户设置新的密码/NTLM 哈希。
    • lsadump::netsync：请求域控制器发送当前和之前的 NTLM 哈希（针对 DC/SRV/WKS）。
    • lsadump::packages：列出所有安全包。
    • lsadump::mbc：获取 MBC（Microsoft Backup Crypto）信息。
    • lsadump::zerologon：执行 Zero Logon 攻击。
    • lsadump::postzerologon：执行 Post Zero Logon 攻击。
  • ts：终端服务模块，用于操作终端服务。
    • ts::multirdp：[实验性] 修补终端服务器服务，以允许多个用户同时连接。
    • ts::sessions：列出或管理终端服务器会话。
    • ts::remote：远程会话管理。
    • ts::logonpasswords：[实验性] 尝试从活动会话中获取登录密码。
    • ts::mstsc：[实验性] 尝试从 mstsc（远程桌面连接）进程中获取密码。
  • event：事件模块，用于处理事件日志。
    • event::drop：[实验性] 修补事件服务以避免记录新事件。
    • event::clear：清除事件日志。
  • misc：杂项模块，包含其他辅助功能。
    • misc::cmd：绕过系统组策略的 DisableCMD 限制，在锁屏/受限环境启动 cmd.exe。
    • misc::regedit：突破 DisableRegistryTools 策略限制，强制启动 regedit.exe。
    • misc::taskmgr：绕过 DisableTaskMgr 组策略，激活被禁用的任务管理器。
    • misc::ncroutemon：针对 Juniper Network Connect VPN 客户端，禁用路由监控功能（常用于隐蔽横向移动）。
    • misc::detours：枚举所有加载模块中的 Detours 式 API 钩子（检测杀软/EDR 的注入行为）。
    • misc::memssp：将 Security Support Provider（SSP）直接注入 lsass 进程内存，绕过文件落地监控。
    • misc::skeleton：提取Windows Hello指纹/面部识别的加密密钥（生物认证绕过）。
    • misc::compress：对提取的凭证数据（如 NTLM 哈希、Kerberos 票据）进行 LZNT1 算法内存压缩。
    • misc::lock：通过内存补丁绕过锁屏界面（需SYSTEM权限）。
    • misc::wp：禁用 PatchGuard 内核保护机制（仅支持 x86 系统）。
    • misc::mflt：操作 Windows 微型过滤驱动（Mini-Filter Driver），用于隐藏恶意文件/注册表项及拦截文件系统操作（如凭证文件读取）。
    • misc::easyntlmchall：生成伪造的 NTLM 挑战响应用于中间人攻击（ADIDNS/LLMNR 投毒）。
    • misc::clip：窃取剪贴板中的密码文本（如管理员粘贴的临时密码）。
    • misc::xor：解密 DPAPI blob 的 XOR 密钥（用于 Credential Manager 凭据导出）。
    • misc::aadcookie：窃取浏览器中缓存的 Azure AD 身份验证 Cookie（云环境横向移动）。
    • misc::ngcsign：破解 Windows Hello 企业版（NGC）的签名密钥（生物特征凭证重用）。
    • misc::spooler：利用 CVE-2021-1675 进行本地提权（需结合 Print Spooler 服务）。
    • misc::efs：利用 EFS 证书缓存漏洞（CVE-2021-36942）窃取加密文件系统的恢复密钥。
    • misc::printnightmare：触发 CVE-2021-34527 远程代码执行（RPC over TCP 端口 135/445）。
    • misc::sccm：利用 SCCM 客户端的 NTLM 中继漏洞（PTH 攻击 Configuration Manager）。
    • misc::shadowcopies：利用 Windows 卷影拷贝服务（VSS）提取历史版本文件（如备份的 SAM/SYSTEM文件）。
    • misc::djoin：伪造 djoin.exe 的域加入请求（用于白银票据攻击）。
    • misc::citrix：窃取 Citrix Workspace 缓存的 SSO 令牌（虚拟化环境凭证重用）。
  • token：令牌操作模块，用于操作和管理令牌。
    • token::whoami：显示当前进程的身份信息（包括用户 SID、权限组等）。
    • token::list：列出系统内存中存储的所有安全令牌（包括其他用户/服务的令牌）。
    • token::elevate：令牌提权功能（可通过窃取SYSTEM等高权限令牌实现权限升级）。
    • token::run：立即执行指定操作（常用于配合其他选项触发提权动作）。
    • token::revert：还原到原始进程令牌状态（用于清除提权操作痕迹）。
  • vault：Windows Vault/Credential 模块，用于管理 Windows 凭证。
    • vault::list：列出系统中存储的 Windows 凭据保管库（Vault）内容，包括保存的登录凭据。
    • vault::cred：提取或操作凭据保管库中的具体凭据数据。
  • minesweeper：扫雷模块，用于扫雷相关操作。
    • minesweeper::infos：获取扫雷游戏进程的内存或数据结构信息。
  • net：网络模块，用于网络相关操作。
    • net::user：管理用户账户（查看、添加、删除域/本地用户，重置密码）。
    • net::group：操作域/本地用户组（查看组成员、添加/删除域管理员等特权账号）。
    • net::alias：管理本地用户别名（如隐藏账户或后门账户的别名配置）。
    • net::session：列出或控制当前系统会话（包括远程登录的域用户会话）。
    • net::wsession：管理远程桌面会话（查询/断开远程终端服务连接）。
    • net::tod：同步或篡改域时间（影响 Kerberos 票据有效期）。
    • net::stats：显示网络服务统计信息（端口、连接状态等）。
    • net::share：管理共享资源（创建/删除共享目录，如隐藏的 ADMIN$ 后门）。
    • net::serverinfo：获取服务器详细信息（操作系统版本、域角色等）。
    • net::trust：操作域信任关系（枚举跨域信任，用于跨域攻击）。
    • net::deleg：管理服务委派权限（查看/配置 SPN 委派）。
    • net::if：管理网络接口（查看 IP 地址、MAC 等网络配置）。
  • dpapi：DPAPI 模块，用于数据保护应用程序接口操作。
    • dpapi::blob：解析 DPAPI 加密的二进制数据块（Blob），通过系统 API 或主密钥（Masterkey）解密原始内容。
    • dpapi::protect：通过 DPAPI 加密指定数据，生成新的加密 Blob。
    • dpapi::masterkey：解密用户主密钥（需密码或域备份密钥），用于解锁 DPAPI 保护的各类敏感数据。
    • dpapi::credhist：结合用户历史密码哈希生成密钥链，破解长期未更换密码的账户主密钥。
    • dpapi::create：伪造主密钥文件（需已知密钥和元数据），用于权限维持或横向移动。
    • dpapi::capi：验证 CAPI 加密密钥的有效性，辅助分析系统加密机制漏洞。
    • dpapi::cng：检查 CNG 密钥存储（如证书私钥），用于提取加密服务账户凭证。
    • dpapi::tpm：尝试绕过 TPM 芯片保护的密钥（需物理访问或 BIOS 权限）。
    • dpapi::cred：解密凭据保管库中的 RDP、共享目录等凭据（需 SYSTEM 权限）。
    • dpapi::vault：提取加密的域用户密码、企业应用登录凭据。
    • dpapi::wifi：一键导出所有连接过的 Wi-Fi 明文密码（路径：%ProgramData%\Microsoft\Wlansvc）。
    • dpapi::wwan：提取 4G/5G 调制解调器的 APN 和认证密钥（需管理员权限）。
    • dpapi::chrome：获取 Chrome 的 Cookies、自动填充密码（需当前用户主密钥）。
    • dpapi::ssh：提取 OpenSSH for Windows 保存的私钥（路径：HKCU\Software\OpenSSH\Agent\Keys）。
    • dpapi::rdg：破解 RDP 连接配置中加密的凭据。
    • dpapi::ps：提取自动化脚本中加密的账号密码。
    • dpapi::luna：解密 LunaHSM 保护的密钥（需 HSM 管理权限或物理攻击）。
    • dpapi::cloudapkd：提取 Azure AD 单点登录（SSO）令牌或密钥（需域管理员权限）。
    • dpapi::cloudapreg：修改或导出 Azure AD 身份验证配置。
    • dpapi::sccm：提取 SCCM 客户端缓存的任务序列、部署凭据（如网络访问账户密码）。
    • dpapi::citrix：解密 Citrix StoreFront 或 XenDesktop 保存的会话凭据（需 Citrix 管理员权限）。
    • dpapi::cache：清除或导出解密缓存，用于隐蔽攻击痕迹。
  • busylight：BusyLight 模块，用于控制 BusyLight 设备。
    • busylight::list：枚举当前系统连接的 BusyLight 设备列表，用于定位目标设备。
    • busylight::status：显示指定 BusyLight 的实时状态。
    • busylight::single：控制 BusyLight 显示固定颜色或模式。
    • busylight::off：关闭 BusyLight 的所有灯光显示，恢复默认状态。
    • busylight::test：触发 BusyLight 自检流程，验证设备响应能力。
  • sysenv：系统环境值模块，用于操作系统环境变量。
    • sysenv::list：显示当前用户或系统环境变量，用于识别可被劫持的路径或敏感配置。
    • sysenv::get：查询特定变量的当前值，辅助分析系统行为或漏洞利用。
    • sysenv::set：添加或篡改变量值，实现持久化或 DLL 劫持攻击。
    • sysenv::del：清除攻击痕迹。
  • sid：安全标识符模块，用于操作和管理 SID。
    • sid::lookup：转换 SID 与对象名称，用于定位高权限账户或服务。
    • sid::query：通过 SID 或名称获取对象的详细属性，辅助信息收集。
    • sid::modify：篡改目标对象的 SID，用于伪装特权身份（需 SYSTEM 或域管理员权限）。
    • sid::add：向目标对象的 sIDHistory 注入伪造 SID，实现权限提升。
    • sid::clear：删除目标对象的 sIDHistory 记录，用于隐蔽攻击痕迹。

  • iis：IIS XML 配置模块，用于操作 IIS 配置。

  • rpc：RPC 控制模块，用于远程控制 mimikatz。
    • rpc::server：在目标系统创建 RPC 端点，用于接收远程指令。
    • rpc::connect：与目标系统的 RPC 服务建立会话，用于后续攻击操作。
    • rpc::close：终止当前 RPC 会话，清理网络痕迹。
    • rpc::enum：列出目标系统开放的 RPC 接口，识别可利用的漏洞或弱配置。
  • sr98：SR98 设备的 RF 模块和 T5577 目标模块。
    • sr98::beep：测试 SR98 设备连接状态，辅助物理渗透中的隐蔽操作。
    • sr98::raw：直接与 RFID 芯片通信，绕过协议限制或破解未知标签格式。
    • sr98::b0：读取或篡改 T5577 标签的厂商代码，用于伪造合法标签绕过校验。
    • sr98::list：扫描并显示附近 RFID 标签的 UID 和类型，定位攻击目标。
    • sr98::hid：克隆或模拟 HID 门禁卡，常用于办公楼物理渗透测试。
    • sr98::em4100：读取/写入 EM4100 标签，通过重放攻击绕过低频 RFID 认证。
    • sr98::noralsy：针对特定欧洲门禁系统的标签破解，需搭配专用天线。
    • sr98::nedap：篡改 Nedap 车辆识别标签，用于绕过停车场闸机或公司车辆管控。
  • rdm：RDM(830 AL) 设备的 RF 模块。
    • rdm::version：获取 RDM-830 AL 的硬件版本与协议兼容性信息，用于判断设备是否支持特定攻击手法。
    • rdm::list：扫描并列出附近 RFID 标签的 UID 及类型，定位可克隆或模拟的目标。
  • acr：ACR 模块，用于 ACR 相关操作。
    • acr::open：建立与读卡器的物理通信（需 USB 直连或蓝牙配对），为后续操作提供通道。
    • acr::close：终止当前会话并释放资源。
    • acr::firmware：获取固件版本或刷写自定义固件（用于绕过协议限制或启用隐藏功能）。
    • acr::info：获取读卡器型号、支持的协议（ISO 14443-A/B）及当前插入卡片的技术选项。
