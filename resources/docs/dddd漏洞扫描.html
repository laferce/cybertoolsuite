dddd漏洞扫描 帮助文档

扫描目标
  • -t, -target string：被扫描的目标。支持多种格式，如 192.168.0.1、192.168.0.0/16、192.168.0.1:80、baidu.com:80，或文件 file.txt（一行一个目标）和 result.txt（fscan/dddd 格式）。
端口扫描
  • -p, -port string：端口设置，默认扫描 Top1000 常见端口。
  • -st, -scan-type string：端口扫描方式。-st tcp 表示 TCP 扫描，-st syn 表示 SYN 扫描，（默认值：tcp）。
  • -tst, -tcp-scan-threads int：TCP 扫描线程数。Windows/Mac 默认为 1000，Linux 默认为 4000（默认值：1000）。
  • -sst, -syn-scan-threads int：SYN 扫描线程数（默认值：10000）。
  • -mp, -masscan-path string：指定 masscan 程序路径，SYN 扫描依赖该工具（默认值：masscan）。
  • -pmc, -ports-max-count int：IP 端口数量阈值。当一个 IP 的端口数量超过此值时，该 IP 将被抛弃（默认值：300）。
  • -pst, -port-scan-timeout int：TCP 端口扫描超时时间（默认值：6；单位：秒）。
主机发现
  • -Pn：禁用主机发现功能（包括 ICMP 和 TCP）。
  • -nip, -no-icmp-ping：当启用主机发现功能时，禁用 ICMP 主机发现功能。
  • -tp, -tcp-ping：当启用主机发现功能时，启用 TCP 主机发现功能。
协议识别
  • -tc, -nmap-threads int：Nmap 协议识别线程数（默认值：500）。
  • -nto, -nmap-timeout int：Nmap 协议识别超时时间（默认值：5；单位：秒）。
探索子域名
  • -sd, -subdomain：开启子域名枚举，默认关闭。
  • -nsb, -no-subdomain-brute：关闭子域名爆破。
  • -ns, -no-subfinder：关闭被动子域名枚举。
  • -sbt, -subdomain-brute-threads int：子域名爆破线程数量（默认值：150）。
  • -ld, -local-domain：允许域名解析到局域网。
  • -ac, -allow-cdn：允许扫描带 CDN 的资产，默认略过。
  • -nhb, -no-host-bind：禁用域名绑定资产探测。
WEB 探针配置
  • -wt, -web-threads int：Web 探针线程数，可根据网络环境调整（默认值：200）。
  • -wto, -web-timeout int：Web 探针超时时间，可根据网络环境调整（默认值：10）。
  • -nd, -no-dir：关闭主动 Web 指纹探测。
HTTP 代理配置
  • -proxy string：HTTP 代理设置。
  • -pt, -proxy-test：启动前测试 HTTP 代理（默认值：true）。
  • -ptu, -proxy-test-url string：测试 HTTP 代理的 URL，该 URL 需返回 200（默认值：https://www.baidu.com）。
网络空间搜索引擎
  • -hunter：从 Hunter 中获取资产，开启此选项后，-t 选项变更为需要在 Hunter 中搜索的关键词。
  • -hps, -hunter-page-size int：Hunter 查询每页资产条数（默认值：100）。
  • -hmpc, -hunter-max-page-count int：Hunter 最大查询页数（默认值：10）。
  • -lpm, -low-perception-mode：Hunter 低感知模式，从 Hunter 直接取响应判断指纹，直接进入漏洞扫描阶段。
  • -oip：从网络空间搜索引擎中以 ip:port 的形式拉取资产，而不是 domain(ip):port。
  • -fofa：从 Fofa 中获取资产，开启此选项后，-t 选项变更为需要在 Fofa 中搜索的关键词。
  • -fmc, -fofa-max-count int：Fofa 查询资产条数，最大为 10000（默认值：100）。
  • -quake：从 Quake 中获取资产，开启此选项后，-t 选项变更为需要在 Quake 中搜索的关键词。
  • -qmc, -quake-max-count int：Quake 查询资产条数（默认值：100）。
输出
  • -o, -output string：结果输出文件（默认值：result.txt）。
  • -ot, -output-type string：结果输出格式，支持 text 和 json（默认值：text）。
  • -ho, -html-output string：HTML 漏洞报告的名称。
漏洞探测
  • -npoc：关闭漏洞探测，只进行信息收集。
  • -poc, -poc-name string：模糊匹配 Poc 名称。
  • -ni, -no-interactsh：禁用 Interactsh 服务器，排除反连模版。
  • -gpt, -golang-poc-threads int：GoPoc 运行线程数（默认值：50）。
  • -ngp, -no-golang-poc：关闭 Golang Poc 探测。
  • -dgp, -disable-general-poc：禁用无视指纹的漏洞映射。
  • -et, -exclude-tags string：通过 tags 排除模版，多个 tags 用逗号连接。
  • -s, -severity string：只允许指定严重程度的模板运行，多个选项用逗号连接，允许的值为：info, low, medium, high, critical, unknown。
配置文件
  • -acf, -api-config-file string：API 配置文件（默认值：config/api-config.yaml）。
  • -nt, -nuclei-template string：指定存放 Nuclei Poc 的文件夹路径（默认值：config/pocs）。
  • -wy, -workflow-yaml string：指定存放 workflow.yaml（指纹 => 漏洞映射）的路径（默认值：config/workflow.yaml）。
  • -fy, -finger-yaml string：指定存放 finger.yaml（指纹配置）的路径（默认值：config/finger.yaml）。
  • -dy, -dir-yaml string：主动指纹数据库路径（默认值：config/dir.yaml）。
  • -swl, -subdomain-word-list string：子域名字典文件路径（默认值：config/subdomains.txt）。
爆破密码配置
  • -up, -username-password string：设置爆破凭证，设置后将禁用内置字典，凭证格式为 'admin : password'。
  • -upf, -username-password-file string：设置爆破凭证文件（一行一个），设置后将禁用内置字典，凭证格式为 'admin : password'。
审计日志、敏感环境必备
  • -a：开启审计日志，记录程序运行日志，收发包详细信息，避免背黑锅。
  • -alf, -audit-log-filename string：审计日志文件名称（默认值：audit.log）。
