功能概述
Golin 工具支持以下功能：
  • 主机存活探测
  • 漏洞扫描
  • 子域名扫描
  • 端口扫描
  • 各类服务数据库爆破
  • PoC 扫描
  • XSS 扫描
  • WebTitle 探测
  • Web 指纹识别
  • Web 敏感信息泄露
  • Web 目录浏览
  • Web 文件下载
  • 等保安全风险问题自查
具体支持：
  • 弱口令/未授权访问：40 余种
  • WEB 组件识别：300 余种
  • 漏洞扫描：XSS、任意文件访问、任意命令执行、敏感信息泄露、默认账户密码等
  • 资产扫描流程：扫描存活主机 -> 判断存活端口 -> 识别协议/组件 -> 基于组件协议进行弱口令、漏洞扫描 -> 输出报告
用法golin.exe [命令] [选项]
命令
  • golin.exe completion：为指定的 shell 生成自动补全脚本。
    • golin.exe completion bash：生成 Bash 的自动补全脚本，需要安装 bash-completion 包。
      • 命令
        • source <(golin completion bash)：当前会话生效。
        • golin completion bash > /etc/bash_completion.d/golin：Linux 平台永久生效。
        • golin completion bash > $(brew --prefix)/etc/bash_completion.d/golin：macOS 平台永久生效。
      • 选项
        • --no-descriptions：禁用补全描述，仅显示命令名称。
    • golin.exe completion fish：生成 Fish 的自动补全脚本。
      • 命令
        • golin completion fish | source：当前会话生效。
        • golin completion fish > ~/.config/fish/completions/golin.fish：永久生效。
      • 选项
        • --no-descriptions：禁用补全描述，仅显示命令名称。
    • golin.exe completion powershell：生成 PowerShell 的自动补全脚本。
      • 命令
        • golin completion powershell | Out-String | Invoke-Expression：当前会话生效。
        • 将上述命令输出添加到 PowerShell 配置文件中：永久生效。
      • 选项
        • --no-descriptions：禁用补全描述，仅显示命令名称。
    • golin.exe completion zsh：生成 Zsh 的自动补全脚本。
      • 命令
        • echo "autoload -U compinit; compinit" >> ~/.zshrc：启用环境变量
        • source <(golin completion zsh); compdef _golin golin：当前会话生效。
        • golin completion zsh > "${fpath[1]}/_golin"：Linux 平台永久生效。
        • golin completion zsh > $(brew --prefix)/share/zsh/site-functions/_golin：macOS 平台永久生效。
      • 选项
        • --no-descriptions：禁用补全描述，仅显示命令名称。
  • golin.exe dirsearch：进行网页目录扫描。
    • --Agent string：自定义请求头的 User-Agent。
    • -c, --chan int：并发请求数量（默认值：100；数值过高可能触发目标防护机制）。
    • --code string：指定需匹配的 HTTP 状态码，多个用逗号分隔（默认值：200）。
    • -f, --file string：自定义字典文件路径（不指定则使用内置字典）。
    • -p, --proxy string：设置代理服务器（例如：http://127.0.0.1:8080）。
    • -t, --timeout int：单次请求超时时间（默认值：3；单位：秒）。
    • -u, --url string：目标URL。
    • --wait int：每次请求后的等待时间（单位：秒）。
  • golin.exe linux：运行 Linux 安全配置核查功能。
    • -c, --cmd string：指定自定义命令文件路径，文件中每行为一条命令。
    • -C, --cmdvalue string：直接输入自定义命令，优先级高于 -c，多条命令用分号分隔。
    • -e, --echo：控制台是否实时输出结果（默认值：False）。
    • -i, --ip string：指定待操作的 IP 地址文件路径（默认值：linux.txt；格式：IP~端口~用户~密码）。
    • -l, --localhost：启用本机采集模式，不连接远程主机（默认值：False）。
    • -s, --spript string：IP 文件中的分隔符（默认值：~；用于分隔 IP、端口、用户、密码）。
    • -v, --value string：指定单个主机信息（格式：IP~端口~用户~密码）。
  • golin.exe mysql：运行 MySQL 安全配置核查功能。
    • -c, --cmd string：自定义 SQL 语句（例如：SELECT * FROM users）。
    • -i, --ip string：指定 MySQL 主机信息文件路径（默认值：mysql.txt；格式：IP~端口~用户~密码~数据库）。
    • -s, --spript string：文件内容分隔符（默认值：~；用于分隔 IP、端口、用户、密码、数据库）。
    • -v, --value string：指定单个主机信息（格式：IP~端口~用户~密码~数据库）。
  • golin.exe oracle：运行 Oracle 安全配置核查功能。
    • -e, --echo：控制台是否实时输出结果（默认值：False）。
    • -i, --ip string：指定 Oracle 主机信息文件路径（默认值：oracle.txt；格式：IP~端口~用户~密码~服务名）。
    • -s, --spript string：文件内容分隔符（默认值：~；用于分隔 IP、端口、用户、密码、服务名）。
    • -v, --value string：指定单个主机信息（格式：IP~端口~用户~密码~服务名）。
  • golin.exe pgsql：运行 PostgreSQL 安全配置核查功能。
    • -e, --echo：控制台是否实时输出结果（默认值：False）。
    • -i, --ip string：指定 PostgreSQL 主机信息文件路径（默认值：pgsql.txt；格式：IP~端口~用户~密码~数据库）。
    • -s, --spript string：文件内容分隔符（默认值：~；用于分隔 IP、端口、用户、密码、数据库）。
    • -v, --value string：指定单个主机信息（格式：IP~端口~用户~密码~服务名）。
  • golin.exe sqlserver：运行 SQL Server 安全配置核查功能。
    • -e, --echo：控制台是否实时输出结果（默认值：False）。
    • -i, --ip string：指定 SQL Server 主机信息文件路径（默认值：sqlserver.txt；格式：IP~端口/实例~用户~密码~数据库）。
    • -s, --spript string：文件内容分隔符（默认值：~；用于分隔IP、端口/实例、用户、密码、数据库）。
    • -v, --value string：指定单个主机信息（格式：IP~端口~用户~密码~数据库 或 IP~实例~用户~密码~数据库）。
  • golin.exe redis：运行 Redis 安全配置核查功能。
    • -i, --ip string：指定 Redis 主机信息文件路径（默认值：redis.txt；格式：IP~端口~[用户]~[密码]）。
    • -s, --spript string：文件内容分隔符（默认值：~；用于分隔 IP、端口、用户、密码）。
    • -v, --value string：指定单个主机信息（格式：IP~端口~[用户]~[密码]）。
  • golin.exe port：进行资产测绘、协议识别和漏洞扫描。
    • -c, --chan int：并发请求数量（默认值：100；数值过高可能导致目标服务崩溃）。
    • -e, --exclude string：排除扫描的端口（如：80,443,8080-8090）。
    • --excludeip string：排除扫描的 IP 文件路径（默认值：noip.txt；每行一个 IP 或网段）。
    • --fofa string：调用 FOFA API 语句（如：app="Apache"）。
    • -i, --ip string：直接指定扫描目标（支持 IP、CIDR、范围格式）。
    • --ipfile string：从文件读取扫描目标（默认值：ip.txt；每行一个IP或网段）。
    • --nocrack：禁用弱口令爆破功能（默认值：False）。
    • --noimg：禁用 Web 服务截图功能（默认值：False）。
    • --noping：禁用 ICMP 存活检测（直接扫描所有 IP；默认值：False）。
    • --nopoc：禁用漏洞扫描（仅做端口探测；默认值：False）。
    • --passwdfile string：自定义弱口令爆破密码字典路径。
    • -p, --port string：指定扫描端口（默认值：0；默认探测TOP 1000端口）。
    • --random：随机打乱扫描顺序（规避流量监测；默认值：False）。
    • -t, --time int：单次请求超时时间（默认值：5；单位：秒）。
    • --userfile string：自定义弱口令爆破用户字典路径。
  • golin.exe route：运行网络设备安全配置核查功能。
    • -c, --cmd string：指定自定义命令文件路径，文件中每行为一条命令。
    • -C, --cmdvalue string：直接输入自定义命令，优先级高于 -c，多条命令用分号分隔。
    • -e, --echo：控制台是否实时输出结果（默认值：False）。
    • -i, --ip string：指定设备信息文件路径（默认值：route.txt；格式：IP~端口~用户、密码/密钥路径）。
    • -s, --spript string：文件内容分隔符（默认值：~；用于分隔 IP、端口、用户、密码/密钥路径）。
  • golin.exe web：以 Web 形式运行。
    • -i, --ip string：绑定Web服务的网络接口 IP（默认值：127.0.0.1；0.0.0.0 表示监听所有接口）。
    • -p, --port string：指定 Web 服务监听的 TCP 端口（默认值：1818）。
    • -s, --save：启用任务结果持久化存储（默认值：False；默认仅内存缓存）。
  • golin.exe windows：读取安全策略并生成 HTML 报告。
    • -h, --help：查看模块帮助信息。
  • golin.exe update：检查程序更新。
    • -p, --proxy string：指定 HTTP/HTTPS 代理服务器地址（格式：http://127.0.0.1:8080 或 https://proxy.example.com）。
  • golin.exe help：获取任何命令的帮助信息。
选项
  • -h, --help：显示 golin 的帮助信息。
